fastf1.core - Core module¶

Contains the main classes and functions.

fastf1.core.REFERENCE_LAP_RESOLUTION = 0.667¶

A distance in meters which indicates the resolution of the reference lap. This reference is used to project car positions and calculate things like distance between cars.

fastf1.core.get_session(year, gp, event=None)[source]¶

Main core function. It will take care of crafting an object corresponding to the requested session. If not specified, full weekend is returned.

Parameters

  • year (number) – Session year

  • gp (number or string) –

    Name or weekend number (1: Australia, …, 21: Abu Dhabi). If gp is a string, a fuzzy match will be performed on the season rounds and the most likely will be selected.

    Some examples that will be correctly interpreted: ‘bahrain’, ‘australia’, ‘abudabi’, ‘monza’.

    Pass ‘testing’ to fetch Barcelona winter tests.

  • event (=None) – may be ‘FP1’, ‘FP2’, ‘FP3’, ‘Q’ or ‘R’, if not specified you get the full Weekend. If gp is ‘testing’ event is the test day (1 to 6)

Returns

Weekend or Session

class fastf1.core.Weekend(year, gp)[source]¶

Bases: object

If you want to handle multiple sessions from the same race event you can use a :class:Weekend instance.

For example you could do the following:

import fastf1 as ff1

weekend = ff1.get_session(2019, 'Monza')
quali = weekend.get_quali() # Q Session
race = weekend.get_race() # R Session

Bases: object

The session class usually will be your starting point. This object will have various information about the event such as name and date. To get the sessions laps use Session.load_laps().

load_laps()[source]¶

With load laps all the timing information is merged into a single pandas dataframe. The first time you run this method on a session it may take a while to load. Multiple megabytes of data have to be downloaded and processed. After that, laps will be stored locally and execution will be much faster.

The returned Laps instance can be used just like a pandas DataFrame with some additional enhancements.

The dataframe columns, therefore, each lap, has the following properties:

  • Time (timedelta): Time when the lap was recorded

  • Driver (string): Three letters driver identifier

  • LapTime (timedelta): Recorded lap time

  • LapNumber (int): Recorded lap number

  • PitOutTime (timedelta): Time when car exited the pit

  • PitInTime (timedelta): Time when car entered the pit

  • Sector1Time (timedelta): Sector 1 recorded time

  • Sector2Time (timedelta): Sector 2 recorded time

  • Sector3Time (timedelta): Sector 3 recorded time

  • SpeedI1 (float): Speedtrap sector 1

  • SpeedI2 (float): Speedtrap sector 2

  • SpeedFL (float): Speedtrap sector 3 (Not sure)

  • SpeedST (float): Speedtrap on longest straight (Not sure)

  • Stint (int): Indicates the stint number

  • Compound (str): Tyre compound name: SOFT, MEDIUM ..

  • TyreLife (int): Laps spent on that compound

  • FreshTyre (bool): Tyre had TyreLife=0 at stint start

  • DriverNumber (str): Car number

  • Team (str): Team name

  • LapStartDate (datetime): When the lap started

  • telemetry (pandas.DataFrame): Telemetry with the following channels:

    • Time (timedelta): Time axis (0 is start of lap)

    • Space (float): Space in meters (from speed and time)

    • Speed (float): Car speed

    • RPM (int): Car RPM

    • nGear (int): Car gear number

    • Throttle (float): 0-100 Throttle pedal pressure

    • Brake (float): 0-100 Brake pedal pressure

    • DRS (int): DRS indicator

    • X (float): GPS X position (normalized)

    • Y (float): GPS X position (normalized)

    • Z (float): GPS Z position (normalized)

    • Status (string): flags OffTrack/OnTrack for GPS

    • SessionTime (timedelta): time elapsed from session start

    • DistanceToDriverAhead (string): distance to next car in m

    • DriverAhead (string): the car ahead

Note

Absolute time is not super accurate. The moment a lap is logged is not always the same and there will be some jitter. At the moment lap time reference is synchronised on the sector time triggered with lowest latency. Expect an error of around ±10m when overlapping telemetry data of different laps.

class fastf1.core.Laps(data=None, index: Optional[Collection] = None, columns: Optional[Collection] = None, dtype: Optional[Union[str, numpy.dtype, ExtensionDtype]] = None, copy: bool = False)[source]¶

Bases: pandas.core.frame.DataFrame

This class wraps Session.laps which is a classic pandas DataFrame with the addition of a few ‘pick’ methods to simplify lap selection and filtering.

If for example you want to get the fastest lap of Bottas you can narrow it down like this:

import fastf1 as ff1

laps = ff1.get_session(2019, 'Bahrain', 'Q').load_laps()
best_bottas = laps.pick_driver('BOT').pick_fastest()

print(best_bottas['LapTime'])
# Timedelta('0 days 00:01:28.256000')

Pick methods will return Laps or pandas Series if only 1 entry is left.

pick_driver(identifier)[source]¶

Select driver given his three letters identifier or its car number:

perez_laps = ff1.pick_driver('PER')
bottas_laps = ff1.pick_driver(77)
kimi_laps = ff1.pick_driver('RAI')
pick_drivers(identifiers)[source]¶

Select drivers given a list of their identifiers. Same as Laps.pick_driver() but for lists:

some_drivers_laps = ff1.pick_drivers([5, 'BOT', 7])
